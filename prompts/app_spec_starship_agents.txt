# Starship Agents - App Specification

## Overview

A WebXR-powered 3D simulation of the USS Enterprise-D bridge (Star Trek: The Next Generation) where real CMD (Command Center) agents are visualized as Starfleet officers at their stations. Data is the acting captain. Agents display live mission status, tool usage, and capability information pulled from the CMD API. Works as an interactive 3D browser experience AND as full immersive VR on Quest 3 via WebXR. This is an April Fools project — fun first, polish second.

---

## Tech Stack

- **Frontend**: Vanilla HTML + Three.js (r160+) + WebXR API
- **Styling**: Inline styles + CSS for HUD overlays
- **3D Engine**: Three.js with WebXR support
- **Backend**: None — reads from CMD API at http://10.0.0.46:3142/api
- **Package Manager**: npm (Vite for dev server + bundling)
- **Deployment**: GitHub Pages (m2ai-portfolio/starship-agents)

## Core Features

### Feature 1: TNG Bridge Environment
**Description**: 3D model of the Enterprise-D main bridge. Doesn't need to be photorealistic — stylized/low-poly is fine. Key structural elements must be recognizable.
**Requirements**:
- Horseshoe bridge layout with captain's chair center
- Science station (rear port), Engineering (rear starboard)
- Ops (front port), Helm (front starboard)
- Tactical rail (standing, rear center)
- Main viewscreen (large flat screen, forward wall)
- Ambient lighting: warm bridge tones (beige/brown panels, blue/red accent lights)
- Use procedural geometry (boxes, cylinders, curved panels) — no external 3D model files needed
- Floor with slightly raised center platform for captain's chair
- Turbolift door (rear, decorative only)

**Test Steps**:
1. Open in browser — bridge geometry is visible and recognizable as TNG bridge
2. Mouse orbit camera works (click-drag to rotate, scroll to zoom)
3. Scene is lit with ambient + point lights

---

### Feature 2: Agent Characters at Stations
**Description**: Simple humanoid figures at each station representing CMD agents. Stylized/abstract is fine — colored uniforms indicate department.
**Requirements**:
- 6 character positions mapped to stations:
  - Captain's chair: "Data" (gold uniform, seated)
  - Science station: "Soundwave" (blue uniform)
  - Engineering station: "Ravage" (gold uniform)
  - Ops station: "Content Agent" (red uniform)
  - Helm station: "Stock Pool" (red uniform)
  - Tactical: "Gap Detection" (gold uniform, standing)
- Characters are simple geometric humanoids (capsule body, sphere head, colored by department)
- TNG uniform colors: Command=red, Operations=gold, Science=blue
- Characters face their stations (Data faces viewscreen)
- Subtle idle animation (slight bob or rotation)

**Test Steps**:
1. All 6 characters visible at correct stations
2. Each has correct uniform color
3. Hovering/clicking a character shows their name

---

### Feature 3: Station Displays (LCARS-style)
**Description**: Each station has a floating display panel showing real-time data from CMD API, styled like LCARS (Star Trek computer interface — rounded rectangles, orange/blue/purple color scheme).
**Requirements**:
- LCARS color palette: #FF9900 (orange), #CC99CC (lavender), #9999FF (blue), #CC6666 (red), #FF9966 (peach), #000000 (background)
- Rounded rectangle frames with characteristic LCARS elbow shapes
- Station displays show:
  - **Captain (Data)**: Mission overview — active count, recent completions, system status
  - **Science (Soundwave)**: Current/last research mission, tools available (WebSearch, WebFetch, firecrawl)
  - **Engineering (Ravage)**: Current/last coding mission, can_spawn_sub_agents flag, build status
  - **Ops (Content Agent)**: Current/last content mission, status
  - **Helm (Stock Pool)**: Number of stock agents loaded, categories
  - **Tactical (Gap Detection)**: Recent capability gaps detected, recommendations
- Displays are Three.js CSS3DRenderer or canvas textures mapped to planes
- Displays update every 5 seconds from CMD API

**Test Steps**:
1. Each station has a visible LCARS-styled display
2. Data's display shows real mission counts from CMD API
3. Soundwave's display shows WebSearch/WebFetch tools
4. Displays refresh with live data

### Feature 4: Main Viewscreen (Mission Feed)
**Description**: The large forward viewscreen shows a live mission feed — scrolling log of recent missions with status, like a ship's operations log.
**Requirements**:
- Full-width screen on forward bridge wall
- Shows last 10 missions with: goal (truncated), status (color-coded), agent assigned, duration
- Status colors: proposed=blue, running=yellow (pulsing), completed=green, failed=red
- New missions animate in from bottom
- LCARS frame around the viewscreen
- "USS ENTERPRISE NCC-1701-D — COMMAND CENTER" header text
- Starfield background behind the text (subtle animated stars)

**Test Steps**:
1. Viewscreen displays recent missions from /api/missions
2. Running missions pulse yellow
3. New missions appear when created via CMD

### Feature 5: WebXR VR Mode
**Description**: "Enter VR" button that switches to immersive WebXR mode for Quest 3.
**Requirements**:
- "Enter VR" button visible in browser mode (bottom-right corner)
- Uses Three.js VRButton helper
- In VR mode:

  - Head tracking for looking around

**Test Steps**:

3. Head tracking works (look around bridge)

### Feature 6: Interactive Camera (Desktop Mode)
**Description**: Orbit camera for desktop/tablet users to explore the bridge.
**Requirements**:
- OrbitControls from Three.js

- Click-drag to orbit, scroll to zoom, right-click to pan
- Click on an agent character to "focus" — camera smoothly flies to their station
- Click on empty space or press Escape to return to default view
- Clicking a focused agent opens a detail panel (HTML overlay) with:
  - Agent name, description, status
  - Full capability list (tools, MCP servers, flags)
  - Current/last mission info

**Test Steps**:
1. Camera orbits smoothly with mouse
2. Clicking Soundwave flies camera to science station

4. Escape returns to default view

### Feature 7: Red Alert Animation
**Description**: When a mission fails, trigger a brief "red alert" — bridge lighting goes red, klaxon sound effect.
**Requirements**:
- Poll /api/missions for status changes
- When a mission transitions to "failed":
  - Bridge ambient lights shift to red for 5 seconds
  - Optional: play a short klaxon sound (web audio API, generate programmatically or use a tiny MP3)

  - "RED ALERT" text on viewscreen
- When a mission completes successfully:
  - Brief green flash on the viewscreen
  - "Mission Complete" text
- Keep it subtle enough to not be annoying on repeat

**Test Steps**:

## Data Models

No local data storage. All state comes from CMD API:

```typescript
// From GET /api/agents
interface Agent {
  id: string;
  name: string;
  description: string;
  skills: string[];
  status: 'available' | 'busy' | 'offline';
  type: 'named' | 'stock' | 'custom';
  active_mission_id: string | null;
}

// From GET /api/agents/capabilities
interface Capabilities {
  agent_id: string;
  tier: number;
  tools: string[];
  mcp_servers: string[];
  can_spawn_sub_agents: boolean;
  max_turns: number;
  timeout: number;
}

// From GET /api/missions
interface Mission {
  id: string;
  goal: string;
  status: 'proposed' | 'running' | 'completed' | 'failed' | 'cancelled';
  agent_id: string | null;
  duration_ms: number | null;
  result: string | null;
}
```

## API Endpoints (CMD — read only)

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/agents | List all agents with status |
| GET | /api/agents/capabilities | All agent capabilities |
| GET | /api/missions | Recent missions (limit 50) |
| GET | /api/missions/:id | Mission detail + logs |
| GET | /api/status | Active/completed summary |
| GET | /api/stats | Outcome statistics |

Base URL: `http://10.0.0.46:3142`

## Constraints & Notes

- **No external 3D models** — everything is procedural geometry. This keeps the bundle tiny and avoids licensing issues.
- **LCARS displays use canvas textures** — render to 2D canvas, map as texture to 3D planes. This is more performant and WebXR-compatible than CSS3DRenderer.
- **API polling, not WebSocket** — CMD doesn't have WS support. Poll every 5 seconds.
- **Fallback mode is essential** — the GitHub Pages URL won't reach the LAN API. Generate realistic fake data so the public demo still looks impressive.
- **April Fools deadline: April 1, 2026** — ship working version by March 31. Polish is secondary to having all 8 features functional.
- **Deploy to GitHub Pages** under m2ai-portfolio org. This produces a public shareable URL.
- **Sound effects**: Generate programmatically via Web Audio API if possible, or include a single tiny MP3 for the klaxon. Don't add a bunch of audio files.
- **Performance target**: 60fps on desktop, 72fps in VR on Quest 3. Procedural geometry + canvas textures should make this easy.
- **init.sh must**: npm install && npm run dev (port 5180)
