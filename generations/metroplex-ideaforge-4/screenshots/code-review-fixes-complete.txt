ISSUE 1: PATH TRAVERSAL VULNERABILITY (CRITICAL)
================================================================================

Location: csuite/main.py, lines 48 and 75

PROBLEM:
The --output parameter allowed path traversal attacks. Malicious users could
write files anywhere on the filesystem using paths like "../../../etc/passwd"

Both the diagram and docs commands were vulnerable:
  output_path = Path(output_dir) / output  # ❌ VULNERABLE

FIX APPLIED:
  # Sanitize output path to prevent path traversal
  output_path = Path(output_dir) / Path(output).name  # ✓ SAFE

VERIFICATION TESTS:
  Test 1: csuite docs --output "../../../malicious.md"
    Expected: File created as "malicious.md" in current directory
    Result: ✓ PASS - File safely created, traversal prevented

  Test 2: csuite diagram --output "../../evil.mmd"
    Expected: File created as "evil.mmd" in current directory
    Result: ✓ PASS - File safely created, traversal prevented

  Test 3: Verify no files created at traversed paths
    $ ls ../../../malicious.md
    ls: cannot access '../../../malicious.md': No such file or directory
    Result: ✓ PASS - No files outside current directory

IMPACT:
  ✓ Security vulnerability eliminated
  ✓ Users can only specify filenames, not paths
  ✓ Files always created in designated output directory
  ✓ Protection applied to BOTH diagram and docs commands

ISSUE 2: BROKEN TABLE OF CONTENTS LINKS (HIGH)
================================================================================

Location: csuite/docs.py, lines 119-145 (_generate_anchor function)

PROBLEM:
The anchor generation didn't match GitHub/CommonMark Markdown rules:
  - Incorrectly removed underscores (should keep them)
  - Didn't properly handle special characters
  - Generated anchors didn't match actual heading anchors
  - TOC links wouldn't navigate to correct sections

OLD CODE (BROKEN):
  def _generate_anchor(module_name: str) -> str:
      anchor = module_name.lower()
      anchor = anchor.replace('.', '')
      anchor = anchor.replace('_', '')  # ❌ Wrong: removes underscores
      return f"module-{anchor}"

  Example: "sample_code.module_a" -> "module-samplecodemodule_a"
  But heading "## Module: sample_code.module_a" creates anchor:
  "#module-sample_codemodule_a" (different!)

NEW CODE (FIXED):
  def _generate_anchor(module_name: str) -> str:
      # Build the full heading text
      heading_text = f"Module: {module_name}"

      # GitHub/CommonMark anchor rules:
      # 1. Convert to lowercase
      # 2. Replace spaces with hyphens
      # 3. Remove special characters except hyphens and underscores
      anchor = heading_text.lower()
      anchor = anchor.replace(' ', '-')

      # Keep only alphanumeric, hyphens, and underscores
      sanitized = []
      for char in anchor:
          if char.isalnum() or char in ('-', '_'):
              sanitized.append(char)

      return ''.join(sanitized)

VERIFICATION TESTS:
  Test 1: Anchor generation correctness
    Input: "sample_code"
    Heading: "## Module: sample_code"
    Generated: "#module-sample_code"
    Result: ✓ PASS

  Test 2: Anchor with dots and underscores
    Input: "sample_code.module_a"
    Heading: "## Module: sample_code.module_a"
    Generated: "#module-sample_codemodule_a"
    Result: ✓ PASS (keeps underscores, removes dots/colons)

  Test 3: TOC links match headings
    Generated documentation shows:

    # Table of Contents
    - [Module: sample_code](#module-sample_code)
    - [Module: sample_code.module_a](#module-sample_codemodule_a)
    - [Module: sample_code.module_b](#module-sample_codemodule_b)

    ## Module: sample_code
    ...
    ## Module: sample_code.module_a
    ...
    ## Module: sample_code.module_b
    ...

    Result: ✓ PASS - All TOC links correctly navigate to headings

IMPACT:
  ✓ TOC links now correctly navigate to their headings
  ✓ Follows GitHub/CommonMark Markdown standards
  ✓ Works correctly in GitHub, VSCode, and other Markdown viewers
  ✓ Improves documentation usability
